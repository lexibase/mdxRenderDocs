O sistema de versionamento do `mdxRenderDocs` permite manter múltiplas versões da documentação de forma organizada e acessível. O processo é **manual**, mas simples e flexível.

## Estrutura de Versionamento

Durante a build do site, o sistema verifica a pasta `app/docs/(versioned)/` e gera automaticamente um arquivo `versions.json` na raiz do projeto. Esse arquivo lista todas as versões disponíveis, da **mais recente para a mais antiga**.

A versão com maior número (a mais recente) é considerada a versão "current" (atual).

### Como criar uma nova versão

1. Crie uma nova pasta dentro de `app/docs/(versioned)/` com o nome no formato:

   ```bash
   version-<numero-da-versao>
   ```

#### Exemplo

```bash
/versioned/version-2.0.0/
```

2. Dentro dessa pasta, crie a estrutura:

```bash
/version-2.0.0/
```

3. Organize os arquivos da mesma forma como `app/docs/`

- Cada subpasta dentro de `/version-<numb>/` representa uma **categoria**.
- Cada arquivo `.mdx` dentro da categoria será uma **página acessível na sidebase**.

## Exemplo de Estrutura

```bash
versioned/
└── version-2.0/
    ├── introducao/
    │   └── boas-vindas.mdx
    └── guia-avancado/
        └── versionamento.mdx
```

## Fluxo

Pegue sua documentação atual (canary) `app/docs/` (pastas como `[version]` e `(versioned)` são excluídas da leitura) e copie para dentro de `(versioned)/version-<x>/` para criar uma nova versão
